## üöÄ How to Host Your Portfolio

### Option 1: GitHub Pages (Recommended - Free & Easy)

1. **Create a GitHub Repository**
   - Go to [github.com](https://github.com)
   - Click "New Repository"
   - Name it: `Amrit004.github.io` (use your GitHub username)
   - Make it Public
   - Click "Create repository"

2. **Upload Your Files**
   - Click "uploading an existing file"
   - Drag and drop these files:
     - `index.html`
     - `styles.css`
     - `script.js`
   - Commit the changes

3. **Enable GitHub Pages**
   - Go to repository Settings
   - Scroll to "Pages" section
   - Source: Deploy from branch
   - Branch: `main` ‚Üí `/root`
   - Click Save

4. **Access Your Site**
   - Your site will be live at: `https://Amrit004.github.io`
   - It may take a few minutes to deploy

### Option 2: Netlify (Free with Custom Domain Support)

1. **Sign up at [netlify.com](https://netlify.com)**
   - Use your GitHub account for easy login

2. **Deploy Your Site**
   - Click "Add new site" ‚Üí "Deploy manually"
   - Drag and drop all three files into the deploy zone
   - Wait for deployment to complete

3. **Your Site is Live!**
   - Netlify will give you a URL like: `random-name.netlify.app`
   - You can change this in Site Settings ‚Üí Domain Management

### Option 3: Vercel (Free & Fast)

1. **Sign up at [vercel.com](https://vercel.com)**
   
2. **Deploy**
   - Click "Add New Project"
   - Import from GitHub or upload files directly
   - Deploy!

3. **Access**
   - Your site will be at: `project-name.vercel.app`

### Option 4: Traditional Web Hosting

If you have a web hosting service (GoDaddy, Bluehost, etc.):

1. Upload all files to your `public_html` or `www` folder via FTP
2. Access via your domain name

## üêõ Troubleshooting

**GitHub repos not showing?**
- Check your internet connection
- Verify GitHub username in `script.js`
- Open browser console (F12) to see any errors

**CSS not loading?**
- Ensure `styles.css` is in the same folder as `index.html`
- Check the file name is exactly `styles.css` (case-sensitive on some servers)

**JavaScript not working?**
- Ensure `script.js` is in the same folder as `index.html`
- Check browser console for errors (F12 ‚Üí Console tab)
